Hybrid Vs. Native

För att börja med en grundläggande skillnad mellan ”Hybrid” & ”Native” ur ett programmeringsperspektiv så är de kodade i olika miljöer där ett ”Native”-app är skrivs i dess naturliga kodspråk, tillhörande ett specifik operativsystem beroende på smarttelefon såsom iOS(Swift/Objektiv-C) samt Android(Java/Kotlin). Tillskillnad från ett ”Hybrid”-app som använder sig av mjukvaruapplikationer som exempelvis ReactNative och Flutter för att skriva kod som går att använda i olika plattformar, detta genom att översätta till plattformens naturliga språk. Skillnader kan märkas beroende på vad applikationen skall användas till men i överlag kan man säga att ”Native” som är kodad till ett specifikt operativsystem har generellt ett snabbare responstid vid mer ”resurskrävande”-appar (3D-rendering) men likaså då appar kräver åtkomst till systemhårdvara (kamera, gps, svepningar osv.). Hybrid kan o andra sida tendera att va långsammare då ett extra ”lager” läggs på när kodningen översätt till dess operativsystem, kan dock nämnda att gapet mellan Native och Hybrid minskar ju längre utvecklingen går framåt. Ett annat perspektiv som framtida utvecklare kan se på den ekonomiska biten att ett ”Native” -app blir dyrare då det krävs minst 2 appar i var sitt naturliga kodspråk(iOS/Android) samt dess underhållning till skillnad från hybrid.
