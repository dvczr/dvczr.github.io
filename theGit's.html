Git

Kort och got så är ”Git” är en versionshanteringssystem som alltså sparar data över de projekt som det arbetas med genom förgreningar (branch) ut av ett huvudstam (master) vilket man kan då arbeta parallellt med huvudstaden för att sedan sammanfoga (merge) med denna, detta gör gör Git genom att lagra snapshots över själva projekten längs med tiden genom att ”commit” den lokala datan för att sedan pusha up den ”commitade” datan mot databasen vilket gör att till skillnad från andra centraliserade versionshanteringssystem så har Git rätt så överlägsens snabbhet tack vare att det är till större del lokal och inte uppkopplad mot en centraliserad server samt för att se de ändringar som gjort så finns de i din dator lokalt. Git använder sig ut av ett mekanism som beräknar en checksumma vilket gör att Git inte tappar data eller att det uppstår korrupt data över tid utan att Git inte märker av det vilket utgörs av ett SHA-1 hash.

GitHub

GitHub kan man då säga är den webbaserade extensionen av Git där versionshistoriken lagras centralt och där man har ett möjlighet att kunna arbeta och ha ett överblick av de olika ”repositories” som har blivit commitade samt pushade upp mot databasen.

GitHub Pages

Siste men inte minst har vi GitHub Pages som har den funktionen för Git och GitHub användare att ha den möjligheten att ta del ut av de statiska webbplatser som hostas för de HTML, CSS och JavaScript filer och som vi grattis användare kan hosta 1 webbplats genom GitHub.io.
